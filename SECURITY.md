# Security Policy

Thank you for helping keep **NGB Platform** and its users safe.

## Supported Versions

Security fixes are applied to the most recent development line and, when applicable, to the latest tagged stable release line.

| Version | Supported |
| --- | --- |
| Latest main branch | Yes |
| Latest stable release | Yes |
| Older releases | No |

## Reporting a Vulnerability

Please **do not open a public GitHub issue** for suspected security vulnerabilities.

Instead, use **GitHub Private Vulnerability Reporting** for this repository:

1. Open the repository on GitHub.
2. Go to **Security**.
3. Open **Advisories**.
4. Click **Report a vulnerability**.

Please include, when possible:

- a clear description of the issue;
- the affected component or area;
- reproduction steps or a proof of concept;
- impact assessment;
- any suggested remediation or mitigation.

## What to Expect

After a private report is received, maintainers will aim to:

- acknowledge receipt within a reasonable time;
- validate and triage the report;
- work on remediation based on severity and impact;
- coordinate disclosure when a fix is available or when mitigations are ready.

Response times may vary depending on report quality, maintainer availability, and the complexity of the issue.

## Disclosure Policy

Please allow time for investigation and remediation before public disclosure.

Coordinated disclosure helps protect users and downstream adopters while a fix is being prepared.

## Scope

This policy applies to vulnerabilities in source code, build and deployment assets, and repository-managed configuration that are part of NGB Platform.

It does not guarantee support for third-party services, self-hosted infrastructure, or deployment-specific customizations outside this repository.
